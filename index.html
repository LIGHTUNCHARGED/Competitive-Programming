Data pipeline

Submission lifecycle

Every accepted solution moves through three phases: ingestion in the browser, processing in GitHub Actions, and presentation on the static frontend.

01 · Trigger

The accepted verdict

You submit a C++ or Python solution on Codeforces. Once the verdict turns accepted, the browser-side extension has everything it needs to capture the source and problem context.

02 · Ingestion

CF-Pusher captures the files

The Chrome extension uses your authenticated Codeforces session, scrapes the solution and README content, then pushes the raw files into the GitHub repository through the REST API.

03 · Processing

GitHub Actions organizes it

The repository push wakes up janitor.yml. It runs organize.py, fetches difficulty data from the Codeforces API, moves files into Sorted_Problems/{Rating}, rebuilds database.json, and commits the updated archive.

04 · Presentation

The dashboard refreshes

GitHub Pages deploys the updated static site. Vanilla JavaScript fetches the latest index and raw repository files to render the dashboard, problem notes, and syntax-highlighted source code.
